Single Partition Regression with direct integration.

The single1d_regression samples the fit of polynomials drawn from a distribution but it is possible to integrate these out directly and avoid the stochastic sampling process. The benefit of this is that the regression is performed quicker. The downside is that results such as credible intervals are unavailable.

Single Partition Regression.

The single1d_regression function performs a regression simulation on a single partition of the given dataset. It will attempt to fit trial polynomials of order 0 to order_max and sampling acceptable polynomials at each order. The method of selecting the order of the polynomial (or the probability distribution of the order of the data) is outlined in [sambridge2006A].

The results are returned in a resultset1d_t structure that can be interrogated with various functions in ::resultset1d.h.

For example of the use of this function, see 1d/single/regression/cubic/cubic.c

Parameters
datasetThe dataset to run the regression on. See the dataset1d_load_known, ::dataset1d_load_estimated, and dataset1d_load_fixed functions.
burninThe number of burn in iterations.
totalThe total number of iterations.
max_orderThe maximum order of the polynomial to allow when fitting a trial curve to the data.
xsamplesThe number of samples to use for discretization of the horizontal axis.
ysamplesThe number of samples to use for discretization of the vertical axis when computing mode, median, and credible intervals.
credible_intervalThe credible interval to use for results expressed as a ratio, i.e. 0.95 for the 95% credible interval.
randomA function pointer to uniform random number generator to use. See the rjmcmc_uniform function.
normalA function pointer to the normal random number generator for generating normally distributed random numbers with 0 mean and a standard deviation of 1. See the rjmcmc_normal function.
resultsA bit mask of results to store. See resultset1d_result_t.
user_callbackAn optional callback function to do your own sampling of the generated curves. Set to NULL if you don't need this.
user_argA user pointer to pass to the callback function.
